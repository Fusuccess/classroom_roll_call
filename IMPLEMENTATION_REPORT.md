## 需求分析

### 用户需求

用户需要能够从 CSV 文件导入班级和学生信息，而不是手动逐个添加。这样可以大大提高工作效率，特别是在班级人数较多的情况下。

### 功能需求

1. 支持从 CSV 文件导入班级和学生信息
2. 支持自定义班级名称
3. 支持数据预览和验证
4. 支持错误处理和用户提示
5. 完全适配 Android 平台

### 非功能需求

1. 性能：导入 1000 个学生应在 10 秒内完成
2. 可用性：用户界面应直观易用
3. 安全性：数据应仅存储在本地
4. 兼容性：支持 Android/iOS/Web

## 实现方案

### 技术选型

1. **file_picker** - 跨平台文件选择
2. **csv** - CSV 文件解析
3. **Flutter** - UI 框架

### 架构设计

```
导入流程：
1. 用户输入班级名称
2. 用户选择 CSV 文件
3. 应用解析 CSV 文件
4. 应用验证数据
5. 应用显示预览
6. 用户确认导入
7. 应用创建班级和学生
8. 应用调用回调函数
9. 应用显示成功提示
```

### 核心模块

1. **ImportExportService**
   - 文件选择
   - CSV 解析
   - 数据验证

2. **ImportClassDialog**
   - 用户界面
   - 数据预览
   - 错误处理

## 问题和解决

### 问题 1：classId 是 final 的

**问题描述：** Student 模型中的 classId 是 final 的，无法在导入后修改。

**解决方案：** 在导入时就生成正确的 classId，并在创建 Student 对象时传入。

**结果：** ✅ 已解决

### 问题 2：导入对话框中的导入逻辑

**问题描述：** 导入对话框需要与导入导出服务协调。

**解决方案：** 在导入对话框中调用导入导出服务的方法，并处理返回的班级和学生对象。

**结果：** ✅ 已解决
