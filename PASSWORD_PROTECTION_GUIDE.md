# 密码保护功能使用指南

## 功能概述

已为应用添加了密码保护层，防止未授权用户访问敏感的学生数据。

## 默认密码

- **默认密码**: `123456`

## 功能说明

### 1. 登录页面
- 应用启动时会显示密码登录页面
- 输入正确密码后才能进入应用
- 登录状态会被保存，刷新页面不需要重新登录

### 2. 修改密码
在设置页面的"安全"部分：
1. 点击"修改密码"
2. 输入旧密码
3. 输入新密码
4. 确认新密码
5. 点击"确认修改"

**注意**: 修改密码后，下次登录需要使用新密码

### 3. 登出
在设置页面的"安全"部分：
1. 点击"登出"
2. 确认登出
3. 返回登录页面，需要重新输入密码

## 部署建议

### 生产环境密码设置

部署到服务器前，**必须修改默认密码**：

1. 本地运行应用
2. 进入设置 → 安全 → 修改密码
3. 将密码改为强密码（建议包含大小写字母、数字、特殊字符）
4. 构建 Web 版本: `flutter build web --release`
5. 部署到服务器

### 密码管理最佳实践

- ✅ 使用强密码（至少8位，包含大小写字母、数字、特殊字符）
- ✅ 定期更换密码
- ✅ 不要在代码中硬编码密码
- ✅ 妥善保管密码，不要分享给不信任的人
- ❌ 不要使用简单密码如 "123456"、"password" 等
- ❌ 不要在浏览器历史记录中保存密码

## 技术实现

### 数据存储
- 密码和认证状态存储在本地 Hive 数据库中
- 使用 IndexedDB（Web 版本）存储
- 数据不会上传到服务器

### 认证流程
1. 用户输入密码
2. 与存储的密码比对
3. 验证成功后设置认证状态
4. 路由层检查认证状态，未认证用户自动重定向到登录页

### 安全性说明
- 这是**应用级别**的密码保护，不是加密存储
- 适合防止普通用户的误操作和非技术性的数据泄露
- 对于高度敏感的数据，建议配合服务器级别的认证（如 HTTPS + 用户账户系统）

## 常见问题

### Q: 忘记密码怎么办？
A: 需要清除浏览器的本地存储数据：
1. 打开浏览器开发者工具（F12）
2. 进入 Application → IndexedDB
3. 找到应用对应的数据库
4. 删除 `auth_box` 表
5. 刷新页面，使用默认密码 `123456` 登录

### Q: 可以在多个设备上使用吗？
A: 可以，但每个设备需要单独输入密码。密码是本地存储的，不同设备之间不同步。

### Q: 部署到服务器后，用户能看到密码吗？
A: 不能。密码存储在用户的浏览器本地存储中，服务器无法访问。

### Q: 支持多用户吗？
A: 当前版本不支持多用户。所有访问应用的人使用同一个密码。

## 进阶配置

### 服务器级别的额外保护（可选）

如果需要更高的安全性，可以在 Nginx/Apache 层添加 HTTP 基本认证：

**Nginx 配置示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    auth_basic "Restricted Access";
    auth_basic_user_file /etc/nginx/.htpasswd;
    
    root /var/www/classroom_roll_call;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

生成 `.htpasswd` 文件:
```bash
htpasswd -c /etc/nginx/.htpasswd username
```

这样用户需要输入两次密码：
1. 服务器级别的 HTTP 基本认证
2. 应用级别的密码认证
