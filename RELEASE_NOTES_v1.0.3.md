## 版本更新

### 新增功能

#### 🔐 登陆与安全功能
- **密码保护**: 应用启动时需要输入密码才能进入系统
- **默认密码**: 123456（部署前必须修改）
- **修改密码**: 用户可在设置中修改访问密码
- **登出功能**: 支持登出，登出后需要重新输入密码
- **状态持久化**: 登陆状态会被保存，刷新页面无需重新登陆

#### 🛡️ 安全特性
- 密码存储在本地 Hive 数据库（Web 版本使用 IndexedDB）
- 应用级别的密码保护，防止未授权用户访问敏感数据
- 支持强密码设置

## 部署建议

### 生产环境部署前必做

1. **修改默认密码**
   - 本地运行应用
   - 进入设置 → 安全 → 修改密码
   - 将密码改为强密码（建议包含大小写字母、数字、特殊字符）

2. **构建 Web 版本**
   ```bash
   flutter build web --release
   ```

3. **部署到服务器**
   - 参考 PASSWORD_PROTECTION_GUIDE.md 中的部署说明

### 可选的额外安全措施

- 在 Nginx/Apache 层添加 HTTP 基本认证
- 使用 HTTPS 加密传输
- 定期更换密码

## 常见问题

### Q: 忘记密码怎么办？
A: 需要清除浏览器的本地存储数据：
1. 打开浏览器开发者工具（F12）
2. 进入 Application → IndexedDB
3. 找到应用对应的数据库
4. 删除 `auth_box` 表
5. 刷新页面，使用默认密码 `123456` 登录

### Q: 可以在多个设备上使用吗？
A: 可以，但每个设备需要单独输入密码。密码是本地存储的，不同设备之间不同步。

### Q: 部署到服务器后，用户能看到密码吗？
A: 不能。密码存储在用户的浏览器本地存储中，服务器无法访问。
